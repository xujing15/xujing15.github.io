By xujing

2017-09-18 更新日期:2017-09-22

hexo+github搭建博客第一篇

安装node：用来生成静态页面的
安装git：把本地的hexo内容提交到github上去
申请github：用来做博客的远程创库、域名、服务器之类的

hexo g #完整命令为hexo generate，生成静态页面至public目录
hexo s #完整命令为hexo server，用于启动服务器，主要用来本地预览（开启预览访问端口（默认端口4000））
hexo d #完整命令为hexo deploy，用于将本地文件发布到github上（将.deploy目录部署到GitHub）
hexo n #完整命令为hexo new，用于新建一篇文章
hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo help #查看帮助
hexo version #查看Hexo的版本

安装HEXO：sudo npm install -g hexo

初始化：创建一个文件夹，进入新建的文件夹中：hexo init
生成静态页面：还是在刚才新建的文件夹中：hexo generate
本地启动：hexo server
还是本地的博客，可以访问http://localhost:4000/

和github关联

建立Repository：建立与你用户名对应的仓库，仓库名必须为【your_user_name.github.io】
配置_config.yml文件：末尾改成（注意冒号后需要加空格）
deploy:
type: git
repository: https://github.com/xujing15/xujing15.github.io.git
branch: master
使用git部署：npm install hexo-deployer-git –save
hexo deploy
可以访问http://xujing15.github.io/

部署步骤

hexo clean
hexo generate
hexo deploy

hexo的配置文件_config.yml

更换主题

下载：git clone https://github.com/huangjunhui/concise.git themes/concise
启用：修改博客根目录下的config.yml配置文件中的theme属性值，将其修改为concise
更新：cd themes/concise;git pull

添加简介

1、在\hexo\themes\light\layout_widget下新建名为intro.ejs的文件，编辑内容
2、在\hexo\themes\light_config.yml中，添加如下代码：
widgets:
intro

hexo搭建博客过程中出现的问题

4000端口被占用：
可以在_config.yml内加上如下代码更改hexo-server运行时的端口号：
server:
port: 4001
compress: true
header: true
其中4001可以换成想要运行的端口号。
